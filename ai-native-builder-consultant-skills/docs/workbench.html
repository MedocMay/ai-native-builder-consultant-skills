核心设计原则

① 整体以 Agent Builder 为主
DL 是场景驱动的可选补充，不是必须。


② 大小模型连用是结论
不是架构预设。有感知任务且感知结果需要被理解，才引入。


③ 先问"该不该做"
场景分析 + 伪需求检查是所有项目的起点。

路径判断
A/B/C/D 四类 → 对应 Agent / Workflow / 规则 / 暂缓


硬依赖
没有 PRD Lite 不设计 Agent；没通过 check_dataset 不训练 DL；没有回滚方案不上线。
