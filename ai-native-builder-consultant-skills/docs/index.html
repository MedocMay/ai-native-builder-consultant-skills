Core Design Principles

六条原则，覆盖所有判断

01
Agent Builder 为主

大多数企业 AI 项目核心是 Agent 范式，DL 是场景驱动的可选补充，不是必须。

02
DL 由场景决定

只有当输入是图像/文档图片/时序数据/语音时，才考虑 DL 小模型。

03
大小模型连用是结论

连用是场景驱动的结论，不是架构预设。两个条件都满足才引入。

04
先问"该不该做"

任何 AI 项目都要先过场景分析和伪需求检查，防止在错误需求上浪费资源。

05
最小化原则

每次只验证一件事，第一版只做最小可用价值，成功标准可以被测量。

06
开源可贡献

Apache 2.0 协议。每个 skill 是独立 markdown，欢迎 PR 补充行业场景与决策框架。
