# Data Foundation Plan
## 基于 AI 的多维度动态材料成本分析系统

**产品：** 材料成本月报 Agent
**Agent 任务类型：** 数据整合 + 生成 + 预测推理
**日期：** 2026-04-13
**版本：** v1.0

---

## 一、现有数据盘点

| 数据源 | 类型 | 颗粒度 | 质量评估 | 可用性 |
|--------|------|--------|---------|--------|
| SAP — 物料成本数据 | 结构化（API） | 物料级 | 高 | ✅ 立即可接入 |
| SAP — BOM / EWO 变更记录 | 结构化（API） | 物料/产品级 | 高 | ✅ 立即可接入 |
| SAP — 实际工缴 vs 标准工缴 | 结构化（API） | 部分在 SAP | 中 | ⚠️ 需与 Excel 补充对齐 |
| 财务经理 Excel（成型） | 结构化表格 | 物料/产品级 | 中-高 | ✅ 结构化入库，工作量可控 |
| 中华商务网价格数据 | 非结构化→结构化 | 品类级（铜铁铝铂钯铑） | 高（PaddleOCR-VL SOTA） | ✅ 现有技术路径可复用 |
| 跨部门数据（采购/研发/工程） | 分散，无统一机制 | 不稳定 | 低 | ❌ 第二阶段推进 |

**冷启动可用数据：** SAP API + 财务经理 Excel + PaddleOCR-VL 价格抓取
**数据缺口：** 跨部门协同数据（第二阶段），取价规则配置（需财务经理标定）

## 二、渐进式数据收集路线

### 第一阶段（现在可启动）

**目标：** AI 接管财务经理能控制的所有数据环节

```
数据源                      收集方式                    落地位置
────────────────────────────────────────────────────────────────
SAP 物料成本/BOM/EWO   →   API 直连，月度自动触发    →   结构化数据库
中华商务网价格          →   PaddleOCR-VL（复用现有）  →   结构化数据库
                            + 规则阈值异常检测
                            + 邮件推财务经理确认
财务经理 Excel          →   一次性结构化入库           →   结构化数据库
                            + 建立标准字段映射表
```

### 第二阶段（协同机制建立后）

**目标：** 打通跨部门数据管道，消除人工汇总环节

```
采购部门采购价          →   统一平台定期上传           →   结构化数据库
研发/工程 BOM 变更      →   系统记录替代邮件通知       →   与 SAP EWO 双向核验
降本计划跟踪数据        →   各业务部门自主填报         →   结构化数据库
（VAVE/采购商务/质量成本）
```

## 四、异常检测规则（Zone B 触发条件）

PaddleOCR-VL 识别准确率高，异常检测聚焦业务逻辑层：

| 检测规则 | 阈值（待财务经理标定） | 触发动作 |
|---------|---------------------|---------|
| 月度价格涨跌幅异常 | ±X%（待定） | 暂停下游计算，邮件推送异常清单 |
| 价格超出历史区间 | 超过近12个月 ±2σ | 标记预警，附历史区间参考 |
| SAP 数据与 Excel 数据差异 | 差异 > Y 元（待定） | 标记差异项，等待财务经理确认以哪个为准 |
| EWO 变更未对应成本变化 | BOM 变更但成本未变 | 主动标注，提示财务经理核查 |
| OCR 识别置信度低 | 置信度 < Z%（待定） | 该字段标记"需人工核实"，不进入计算 |

**⚠️ 重要：** 以上阈值 X、Y、Z 必须在上线前由财务经理与技术团队共同标定，未标定前外部价格抓取不进入 Zone A 自动执行。

## 五、标注策略（Zone B 确认即标注）

**核心原则：** Zone B 的每次财务经理确认，自动成为一条高质量标注数据，零额外工作量。

| 标注来源 | 内容 | 标注者 | 数据价值 |
|---------|------|--------|---------|
| 价格异常确认 | 异常价格是否采用 / 修正值 | 财务经理 | 阈值校准数据 |
| 推理 SOP 节点确认 | 每个推理节点是否认可 | 财务经理 | 预测模型反馈数据 |
| 差异原因修改 | 财务经理对 AI 标注的修改内容 | 财务经理 | 语言表达优化数据 |
| EWO 变更核查结果 | 变更对成本的实际影响 | 财务经理 + 工程 | BOM 分析准确性数据 |

**标注数据捕获要求（系统设计阶段必须规划）：**
```json
{
  "id": "confirm_20260415_copper_price",
  "type": "zone_b_confirmation",
  "input": { "ocr_value": 68500, "source": "中华商务网", "date": "2026-04-15" },
  "agent_flag": "anomaly_monthly_change_+8.3%",
  "human_decision": "confirmed / modified",
  "modified_value": null,
  "annotator": "财务经理",
  "timestamp": "2026-04-15T10:30:00",
  "split": "training_candidate"
}
```

## 六、评估集构建计划

**划分原则：** 按月度报告期划分，不按随机划分。同一月份的数据不跨训练集和评估集。

| 类型 | 数量目标 | 来源 | 覆盖场景 |
|------|---------|------|---------|
| 正常月报路径 | 6 份（历史月报） | 财务经理历史 Excel + 历史月报 | 价格平稳、无 EWO 变更的标准流程 |
| 价格异常场景 | 3-5 份 | 历史异常月份 + 人工构造 | 有色金属大幅波动月份 |
| EWO 变更场景 | 3-5 份 | SAP 历史 EWO 记录 | BOM 变更对成本的传导 |
| 降本计划落地场景 | 2-3 份 | 历史 VAVE/采购商务记录 | 降本措施对成本的实际影响 |

**评估集更新机制：**
- 每季度评审一次：移除已修复的失败模式，加入新发现的边界案例
- 触发提前更新：发现新的系统性失败模式时

## 七、数据治理

| 维度 | 决策 |
|------|------|
| 存储位置 | 内部服务器 / 企业云（待技术选型确认） |
| 访问权限 | 财务经理：读写 / 技术团队：读 / 其他部门：仅上传自己数据 |
| 数据保留期 | 原始数据保留 3 年（匹配财务审计周期） |
| 敏感信息处理 | 供应商具体价格数据不对外共享，内部分级访问 |
| 训练集/评估集分离 | 物理分离，不同目录，评估集只通过 eval 脚本访问 |
| 版本控制 | 每次月报发布记录数据集版本号，支持事后追溯 |

## 八、冷启动路径（第一阶段启动计划）

- [ ] **Week 1：** 财务经理 Excel 结构化入库，建立标准字段映射表
- [ ] **Week 1：** SAP API 接口权限申请，字段对齐验证

## 九、最大风险项（上线前必须解决）

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| 阈值未标定 | 异常检测规则没有具体数值，Zone B 无法正常触发 | 上线前必须完成标定，这是硬前提 |
| SAP 字段与 Excel 不一致 | 同一数据在两个来源的口径不同 | Week 1 建立字段映射表时逐一核对 |
| 取价日规则未配置 | 不知道每月取哪一天的价格 | 财务经理在系统配置中明确取价规则后才触发抓取 |
| 第二阶段协同阻力 | 其他部门不配合数据上传 | 渐进方案已规避，第一阶段不依赖跨部门数据 |
