## 案例 1：材料成本分析系统（制造业 / 财务 SaaS）

**场景：** 某汽车零部件公司财务团队希望用 AI 自动化每月的材料成本分析报告。原流程：财务助理花 3 天从 SAP 导出数据 + 收集各部门 Excel + 手工汇总，财务经理审核后发管理层。

**SOP 走过：** 阶段 0 (场景分析 + 伪需求检查) → 阶段 1 (项目定义 + PRD Lite) → 阶段 2 (Agent 边界 + SOUL + 数据基础 + 技术选型) → 阶段 3 (上线评审 + 飞轮设计)

**结论：** A 类（Agent 主路径），不需要 DL，但需要 OCR 处理供应商发票（小模型补充）。

| `材料成本分析系统_迭代飞轮设计.md` | `iteration-flywheel` | 每周复盘节奏 + 四个改进杠杆 |

### 关键决策

1. **不用通用大模型 API** — 数据合规要求，财务数据不能出内网，必须本地部署
2. **小模型负责感知，大模型负责理解** — 供应商发票图片用 PaddleOCR-VL 提取结构化数据，文心 4.5-28B 做异常解释和报告生成
3. **Zone B 主导** — 月报作为对外正式产出，AI 生成草稿，财务经理逐节点确认推理过程
4. **冷启动两个月** — 用 50 份历史月报作为 few-shot 示例，前两个月每份月报由经理修改后 fine-tune

## 怎么使用这些案例

**作为模板：** 把 SOUL.md 拷到自己的项目里，替换业务部分（保留五层结构）

**作为参考：** 看交付物长什么样，知道每个 skill 应该输出什么粒度的内容

**作为证据：** 给业务方解释"AI 咨询能产出什么"时，直接展示这些文档
