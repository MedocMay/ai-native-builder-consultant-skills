# Pseudo-AI Detection — 伪 AI 需求识别

## 咨询链位置

**在 SOP 中：** Module 0.5，在场景分析（`scene-analysis`）之后、项目定义（`ai-project-definition`）之前。

**触发条件：** 以下任何一种情况出现时，必须先过此 Skill：
- 需求描述像愿景而不像项目
- 需求边界不清晰
- 不确定第一版能验证什么
- 场景分析结论是 B/C/D 类但仍想推进

**常见联动：**
- 通过检查（不是伪需求）→ `ai-project-definition`
- 需要缩小重写 → 重新过 `scene-analysis`
- 当前不建议推进 → 说明理由，记录存档，等条件具备

---

## 八项检查清单

对照以下八项逐一判断，"是"越多，伪需求概率越高：

| 检查项 | 判断 | 说明 |
|--------|------|------|
| **1. 没有明确用户** | 是 / 否 | 谁会用这个 AI？说不清楚就没有用户 |
| **2. 没有具体任务** | 是 / 否 | "提高效率"不是任务，"把 X 步骤从 3 天缩短到 1 小时"才是 |
| **3. 写得像愿景** | 是 / 否 | "打造智能化平台"是愿景，"第一版让财务经理每月少花 4 小时"是项目 |
| **4. 第一阶段无法验证** | 是 / 否 | 6 个月内看不到可测量的效果，就是过大的需求 |
| **5. 更像流程问题** | 是 / 否 | 如果人工流程本身就乱，AI 会让混乱更快发生 |
| **6. 更像数据治理问题** | 是 / 否 | 数据不干净、不可用，加 AI 只是在垃圾上盖房子 |
| **7. 自己推不动** | 是 / 否 | 需要 5 个部门配合但只有 1 个部门在推，条件不具备 |
| **8. 做错风险过高** | 是 / 否 | AI 犯错会导致财务错误、安全事故、法律风险等 |

---

## 判断结果与行动

### 结果 1：不是伪需求，继续推进
**条件：** 八项中"是"不超过 1 项，且不涉及第 5、6、7 项

**行动：** 进入 `ai-project-definition`，开始正式项目定义

---

### 结果 2：方向对，但要缩小重写
**条件：** 第 3 或 4 项为"是"，但其他项基本通过

**常见症状：**
- 描述的是一个大系统，而不是一个具体功能
- 第一版包含了太多"以后再说"的功能
- 成功标准太模糊（"用户满意度提升"）

**行动：**
1. 把需求切割：只保留第一版能独立验证的最小部分
2. 重新明确：这一版解决谁的什么具体问题
3. 重写后重新过此清单

**缩小方向示例：**
```
原需求：建设智能制造成本管理平台
缩小后：第一版只做"月度材料成本变动自动分析报告"
         目标用户：财务经理（1人）
         成功标准：每月报告生成时间从3天→2小时
```

---

### 结果 3：当前更适合非 AI 优先解
**条件：** 第 5 或 6 项为"是"

**非 AI 优先解方案：**

| 问题类型 | 建议方案 |
|---------|---------|
| 流程不规范 | 先写标准 SOP，固化流程，再考虑 AI |
| 数据不干净 | 先做数据治理（清洗、标准化、归集），再考虑 AI |
| 权责不清 | 先明确谁负责什么，再考虑 AI 介入哪一步 |
| 规则可以描述 | 直接写代码实现（Vibe Coding），比 AI 更稳定 |

**关键判断：** 如果 AI 下线，业务能不能继续跑？如果不能，说明前置条件没有具备。

---

### 结果 4：当前不建议推进
**条件：** 第 7 或 8 项为"是"，或"是"超过 4 项

**说明理由（必须明确写出）：**
- 缺少的条件是什么？
- 什么时候这些条件可能具备？
- 在等待期间可以做什么准备？

**记录存档：** 把判断结果和理由记录下来，条件具备时重新评估。

---

## 高风险伪需求模式识别

以下是最常见的伪 AI 需求模式，遇到立刻警惕：

**模式 1："AI 赋能 XX 业务"**
"赋能"是宣传语不是需求。问：具体哪个环节？谁的哪个操作？

**模式 2："建一个知识库，把所有文档放进去"**
知识库不是需求，是工具。问：谁会用这个知识库？用来做什么具体的任务？

**模式 3："做一个智能助手，让员工问任何问题"**
范围无限大意味着没有重点。问：最高频的 3 类问题是什么？先只做这 3 类。

**模式 4："把现有系统接上 AI"**
接入不是目的。问：接上之后用户的哪个具体操作会变得不同？

**模式 5："数据都有了，拿去训练 AI"**
有数据不等于数据可用。问：数据有没有标注？质量如何？覆盖哪些场景？
